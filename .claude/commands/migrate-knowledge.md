# 지식저장소 마이그레이션

사용자가 `$ARGUMENTS`(원본 폴더 경로 + 선택적 메모)를 지식저장소로 통합하려 한다. 아래 절차를 끝까지 자율 수행하라. **원본 폴더는 절대 수정하지 않는다.** 시작 전에 `Research-Vault/_wiki/WIKI-RULES.md`와 `Research-Vault/_wiki/INDEX.md`를 읽는다.

> 다른 폴더에서도 이 명령을 쓰려면 이 파일을 사용자 레벨(`~/.claude/commands/`)로 복사하고 아래 경로들을 지식저장소 절대경로 기준으로 바꾼다.

## 0. 입력 파싱
- `$ARGUMENTS`에서 **원본 폴더 경로**를 추출. 없거나 폴더가 없으면 한 번 묻고 중단.
- 나머지는 **주제 힌트·선별 범위**로 해석 (예: "report 폴더만"). 없으면 폴더를 보고 직접 판단.

## 1. 폴더 훑어보기 + 선별
- 재귀 탐색해 확장자·크기·하위구조 파악.
- **가져올 것**: 최종 보고서(.md/.docx/공개판 PDF), 케이스·소스 노트, 정리된 분석.
- **버릴 것**: 코드, 중간 산출물, 임시파일(tmp_·_old·.bak), 빌드 캐시, 대용량 비공개 원본 PDF(공개판 있으면 공개판만).
- 선별 결과를 한 줄 보고하고 진행 (되돌리기 쉬우므로 승인 대기 안 함).

## 2. 복사
- 대상: `Research-Vault/Project-Knowledge/<프로젝트슬러그>/` (kebab-case).
- 하위 구조 보존. PDF 포함하려면 `.gitignore`에 `!Research-Vault/Project-Knowledge/**/*.pdf` 예외 확인·추가.

## 3. 위키 통합 (핵심)
- 복사 자료를 **에이전트 병렬**로 읽어 핵심 주장(수치 포함)·데이터 출처 추출 (PDF는 pages로 20p씩).
- INDEX.md의 기존 개념 페이지와 대조:
  - **흡수 가능** → 해당 `_wiki/concepts/<slug>.md`를 WIKI-RULES §2에 따라 갱신 (모순은 ⚔️ 양쪽 보존, 수치표 보강, Living Summary·frontmatter 갱신).
  - **새 주제** → 새 개념 페이지 신설 (WIKI-RULES §2 형식). 25개 상한 의식.
- 기존 위키와 **교차 연결** (`[[링크]]`).

## 4. 데이터 레지스트리
- 반복 조회 가능한 데이터 출처만 골라 `_data-registry/sources/<slug>.md` 추가(있으면 인용 노트 보강)하고 `DATA-SOURCES.md`·`sources_count` 갱신. 일회성 기사·보도자료·법원문서·기술 스펙 제외.

## 5. 인덱스·장부 갱신
- `_wiki/INDEX.md`: 새/갱신 페이지 반영, 카운트·"최근 갱신" 갱신.
- `_wiki/INGEST-LOG.md`: 통합 파일을 `- [[파일명]] (통합: <오늘> → [[개념페이지]])`로 추가, `ingested_count` 갱신.

## 6. 커밋·푸시 (git 연동 시)
- `git add -A` 후 PDF 추적 확인(`git ls-files`)하고 커밋·푸시.

## 7. 보고
완료 후: 만든/갱신한 개념 페이지, 등록한 데이터 출처, 복사 파일 수, (있으면) 커밋 해시, 교차 연결·미해결 질문을 간결히 보고.
